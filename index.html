My Story So Far

The decision to learn to code was initially undertaken in response to a simple need: to automate repetitive data-entry tasks in an academic project focused on music theory. I asked a computer scientist friend for advice, and was given the straightforward recommendation to "just learn C". Taking their suggestion to heart, I immediately purchased The C Programming Language by Brian Kernighan and Dennis Ritchie, and started reading and coding along. However, as I delved deeper, I realized that coding is much more than just automation. It is a powerful tool for solving complex problems and, more broadly, innovating. I have always been a believer in lifelong learning, and coding has given me a new avenue to explore my passion for problem-solving.

Something that struck me from the very start was how much can be accomplished with so little. After barely two weeks of starting to learn C, and having only finished reading the first chapter and a half of the K&R C book, I was already able to design and implement my first original, useful program, with barely any knowledge beyond for and while loops, and basic arithmetic operations and conditional expressions. After barely two months, while I still considered myself an absolute beginner at JavaScript, I was able to make a fully interactive web app with procedurally generated animations and audio.

I am unbelievably excited to see what becomes possible as I continue to learn and grow as a developer, and to work alongside and learn from other developers!

2x2 Web App...

This project combined the DOM-manipulation techniques I learned from making this project gallery with my previous, terminal-based program to make a fully interactive web app to explore 2x2 canonic sequences. It will hopefully make a nice educational resource for musicians with an interest in theory.

The set of all 2x2 canonic sequences is generated algorithmically in JavaScript, and populates a column to the left-hand side of the screen. To the right, details about individual patterns the user clicks on are displayed, and above is a dynamic menu by which they can be filtered. Information about each pattern is also dynamically generated via various algorithms, and even the images were originally compiled from algorithmically generated LilyPond code, before being replaced with my own custom canvas-based notation renderer. I also added dynamically-generated audio using the Web Audio API.

Over 1,400 lines of JavaScript and counting! ü•≥
